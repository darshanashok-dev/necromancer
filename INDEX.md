## üìù Documentation Standards

### File Naming
- Use UPPERCASE for main docs (README.md, ARCHITECTURE.md)
- Use lowercase for subdirectory docs (agents/coroner/README.md)
- Use hyphens for multi-word files (IMPLEMENTATION_PLAN.md)

### Formatting
- Use Markdown format
- Include emoji for visual hierarchy
- Use code blocks for examples
- Include diagrams where helpful
- Add table of contents for long docs

### Structure
- Start with title and purpose
- Include overview section
- Use clear headings
- End with future enhancements
